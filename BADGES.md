# Yggdrasil Badge System

Badges are community-verifiable markers of progress. They are not issued by a central authority — they are earned by doing the work and proved by your public commit history. Any hiring manager, peer, or collaborator can verify your badges by looking at your portfolio repository.

---

## How Badges Work

1. **Complete the requirements** for a badge (detailed below)
2. **Commit all required work** to your public portfolio repo
3. **Update your PROGRESS.md** with the badge and date earned
4. **Post in Discord #badge-claims** with a link to your repo — community members verify and celebrate

There is no certificate authority. There is no PDF. There is a GitHub repo full of working code that proves you can do the thing. That is more valuable.

## Specialty Badges

In addition to phase completion badges, the following specialty badges recognize outstanding work in specific areas.

### 🔬 Paper Replicator
Replicate 3 ML papers from scratch, each with a public reproduction report.

### 🧮 Math Purist
Implement 10 algorithms from scratch (no ML libraries) across any phase.

### 🚀 Speed Runner
Complete any full phase in under 4 weeks while maintaining all quality standards.

### 🤝 Community Pillar
Merge 5 accepted contributions to the Yggdrasil curriculum.

### 🏆 Benchmark Beater
Beat the project baseline by a statistically significant margin in any module project.

### 📖 Explainer
Write a lesson or tutorial that is accepted into the Yggdrasil curriculum.

## A Note on Verification

Every badge requirement maps to code in a public GitHub repo. This is deliberate. Badges without verifiable evidence are worthless. A repo full of working code that someone built themselves is evidence.

If you share your badge on LinkedIn or a resume, link your portfolio repo. If a hiring manager asks "can you prove it?" — you can.

That is the point.
